In this task, you'll add a new feature to an existing script using two approaches: direct modification and duplication. Then, you'll compare the results and submit your code for checking.

## 🗂️ Files you'll need

Check that `duplication_challenge` has been added to your GitHub account. You'll need the following files:

- `sales_report_generator.py`
- `sample_sales_data.csv`

## **🔍 Step 1: Analyze the existing code**

1. Open `sales_report_generator.py` in Cursor.
2. Explore the file with Cursor's help. Try these prompts:
    
    <aside>
    
    What does this sales report generator do?
    
    </aside>
    
    <aside>
    
    What would I need to change to add 3-month sales forecasting?
    
    </aside>
    

## **✏️ Step 2: Direct modification**

First, try modifying the file directly. Ask Cursor to add 3-month sales forecasting to `sales_report_generator.py`.

You don't need to fix any errors at this stage — just accept the changes and save the updated file.

## 🧩 Step 3: Duplication

Now, try a more structured approach with duplication.

1. Ask Cursor to write a Markdown plan:
    
    <aside>
    
    I need to add 3-month sales forecasting. Write a Markdown plan for what `sales_report_generator.py` should do differently. Save the output as `sales_forecast_generator_plan.md`.
    
    </aside>
    
2. Use the Markdown plan to implement a new version of the script. Name it `sales_forecast_generator.py`.

## ⚖️ Step 4: Compare both versions

Ask Cursor: 

<aside>

Compare `sales_report_generator.py` and `sales_forecast_generator.py`. How are they different? Which one has cleaner code?

</aside>

## 📤 Step 5: Submit your work

Submit `sales_forecast_generator.py` and `sales_forecast_generator_plan.md` for checking. 

---


## **✅ Success criteria**

Make sure that your code:

- [ ]  Includes logic for 3-month sales forecasting
- [ ]  Follows the plan outlined in `sales_forecast_generator_plan.md`
